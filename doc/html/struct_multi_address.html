Detailed Description

Normally, addresses have been represented using string addresses, like:

tcp4://127.0.0.1:1234 udp4://10.20.30.40:5060 ws://1.2.3.4:5678 tcp6://[1fff:0:a88:85a3::ac1f]:8001 This isn't optimal. Instead, addresses should be formatted so:

Binary format:

(varint proto><n byte="" addr>="">)+ <1 byte ipv4 code><4 byte ipv4 addr><1 byte udp code><2 byte udp port> <1 byte ipv6 code><16 byte ipv6 addr><1 byte tcp code><2 byte tcp port>
