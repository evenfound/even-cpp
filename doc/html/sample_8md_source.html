1 # QNanoPainter
2 
3 QNanoPainter is an OpenGL accelerated C\++ vector drawing library for Qt, offering optimal performance, productivity and rendering quality all-in-one. QNanoPainter API is a mixture of QPainter and HTML5 canvas APIs. In other words, it's very close to HTML5 canvas but transformed to object oriented C++ with separate classes.
4 
5 QNanoPainter can be used together with Qt5 UIs in different ways:
6 
7 * Use QNanoQuickItem and QNanoQuickItemPainter when implementing custom C\++ Qt [QQuickItems](http://doc.qt.io/qt-5/qquickitem.html). This is comparable to [QQuickPaintedItem](http://doc.qt.io/qt-5/qquickpainteditem.html), but instead of QPainter offers own QNanoPainter API for drawing.
8 * Use QNanoWidget when implementing custom QWidget. This is based on [QOpenGLWidget](http://doc.qt.io/qt-5/qopenglwidget.html).
9 * Use QNanoWindow when implementing single-view UI, optimal for embedded usage. This is based on [QOpenGLWindow](http://doc.qt.io/qt-5/qopenglwindow.html).
10 
11 QNanoPainter uses excellent [NanoVG](https://github.com/memononen/nanovg) as its rendering backend.

20 ## Features
21 
22 Here is a non exhaustive list of QNanoPainter features:
23 
24 * **Vector drawing**: Path-based drawing of rectangles, circles, lines etc. filled and/or stroked. Brush can be color, gradient or image pattern. Different cap and join options for stoked lines.
25 * **Images**: Transformed images or rectangular areas of images. Mipmapping for smoother rendering of scaled images.
26 * **Text**: Contains set of default fonts and supports loading custom truetype fonts. Text aligning and wrapping. Letter-spacing and blur for shadow/glow effect.
27 * **Antialiasing**: Adjustable antialiasing amount for vector drawing.
28 * **Pixel aligning**: Drawing and texts can be either freely positioned (smooth animations) or aligned to pixels (sharp rendering).
29 * **Extra**: Supports high-DPI retina resolutions, global alpha, transforms, painter states etc.

32 ## Usage
33 
34 Taking QNanoPainter into use in your Qt application is simple:
35 
36 * Copy 'libqnanopainter' directory into your project. Sources are included directly instead of building separate static/dynamic library which makes usage simple and allows easier customization of QNanoPainter itself.
37 * In your project .pro file, include QNanoPainter with something like:
38 
39 
40 ```
41 include(src/libqnanopainter/include.pri)
42 ```
43 ### For custom QQuickItem
44 
45 * Implement your QNanoQuickItem and QNanoQuickItemPainter based classes (see more about these below or from available examples).
46 * Export your item into QML in main.cpp with something like:
47 
48 ```
49 qmlRegisterType<MyQNanoItem>("MyQNanoItem", 1, 0, "MyQNanoItem");
50 ```
51 
52 * Use your item in QML:
53 
54 ```
55 import QtQuick 2.4
56 import MyQNanoItem 1.0
57 
58 Item {
59  ...
60  MyQNanoItem {
61  anchors.fill: parent
62  }
63 }
64 ```
65 
66 To create own QNanoPainter item you should implement 2 classes:
67 
68 * **QNanoQuickItem**: This will be the visual QQuickItem providing API towards QML side. Your painter will be created and attached to item in *createItemPainter()* method.
69 * **QNanoQuickItemPainter**: This will handle the drawing using QNanoPainter API in the *paint()* method. Data with the item will be transferred in *synchronize()* method.
70 
71 The main reason for two classes is that these will run in separate threads for optimal performance. And because of this, all communication between these two need to happen in *synchronize()* method.
72 
73 Here is a complete HelloWorld example item:
74 
75 ```C++:
76 #include "qnanoquickitem.h"
77 #include "qnanoquickitempainter.h"
78 
79 // HelloItemPainter contains the painting code
80 class HelloItemPainter : public QNanoQuickItemPainter
81 {
82  Q_OBJECT
83 
84 public:
85  HelloItemPainter()
86  {
87  }
88 
89  void paint(QNanoPainter *p)
90  {
91  // Paint the background circle
92  p->beginPath();
93  p->circle(width()*0.5, height()*0.5, width()*0.46);
94  QNanoRadialGradient gradient1(width()*0.5, height()*0.4, width()*0.6, width()*0.2);
95  gradient1.setStartColor("#808080");
96  gradient1.setEndColor("#404040");
97  p->setFillStyle(gradient1);
98  p->fill();
99  p->setStrokeStyle("#202020");
100  p->setLineWidth(width()*0.02);
101  p->stroke();
102  // Hello text
103  p->setTextAlign(QNanoPainter::ALIGN_CENTER);
104  p->setTextBaseline(QNanoPainter::BASELINE_MIDDLE);
105  QNanoFont font1(QNanoFont::DEFAULT_FONT_BOLD_ITALIC);
106  font1.setPixelSize(width()*0.08);
107  p->setFont(font1);
108  p->setFillStyle("#B0D040");
109  p->fillText("HELLO", width()*0.5, height()*0.4);
110  // QNanoPainter text
111  QNanoFont font2(QNanoFont::DEFAULT_FONT_THIN);
112  font2.setPixelSize(width()*0.18);
113  p->setFont(font2);
114  p->fillText("QNanoPainter", width()*0.5, height()*0.5);
115  }
116 };
117 
118 // HelloItem provides the API towards QML
119 class HelloItem : public QNanoQuickItem
120 {
121  Q_OBJECT
122 
123 public:
124  HelloItem(QQuickItem *parent = 0)
125  : QNanoQuickItem(parent)
126  {
127  }
128 
129  // Reimplement
130  QNanoQuickItemPainter *createItemPainter() const
131  {
132  // Create painter for this item
133  return new HelloItemPainter();
134  }
135 };
136 ```
