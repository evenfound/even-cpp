98 // ipfs_path_clean_path returns the shortest path name equivalent to path
99 // by purely lexical processing. It applies the following rules
100 // iteratively until no further processing can be done:
101 //
102 // 1. Replace multiple slashes with a single slash.
103 // 2. Eliminate each . path name element (the current directory).
104 // 3. Eliminate each inner .. path name element (the parent directory)
105 // along with the non-.. element that precedes it.
106 // 4. Eliminate .. elements that begin a rooted path:
107 // that is, replace "/.." by "/" at the beginning of a path.
108 //
109 // The returned path ends in a slash only if it is the root "/".
110 //
111 // If the result of this process is an empty string, Clean
112 // returns the string ".".
113 //
114 // See also Rob Pike, ``Lexical File Names in Plan 9 or
115 // Getting Dot-Dot Right,''
116 // https://9p.io/sys/doc/lexnames.html

130

132  if (!path) {

134  return NULL;
135  }

137
